## Compliance Boundary

This report does not claim final execution-manual compliance. Fallback, synthetic, demo, and static paths remain visible as limited paths and cannot be used as proof that the execution manual is fully satisfied.
Azure Document Intelligence OCR confidence, bbox, and table_blocks must come from Azure raw responses and are not synthesized by the adapter. Azure F0 can be used for small local validation, but its page and rate limits do not replace final real-sample verification.
The Azure OCR real image validation recorded only a safe summary. `.env` was not committed, API keys were not recorded, the `local_storage` receipt sample was not committed, and the full Azure raw response was not stored in this report.
Manual OCR dataset results with `is_production_evaluation=false` are marked as non-production manual acceptance, not production quality claims.
The successful manual OCR run used one public sample only. `.env`, API keys, and the `local_storage` image were not committed, and the result must not be interpreted as production-scale Evaluation coverage.
Manual classification dataset support uses synthetic text samples and records `is_production_evaluation=false`. The successful manual classification run used six synthetic samples with `accuracy=1.0`, `macro_f1=1.0`, and no failed cases, but it is not a production-scale classification evaluation and does not exercise the full uploaded-document DB workflow.
Manual extraction dataset support uses synthetic text/OCR-block samples and records `is_production_evaluation=false`. The current dataset includes a `require_source_bbox=true` fixture so field and line-item source bbox can be validated without calling a real LLM. It is not a production-scale extraction evaluation or real Provider quality claim.
Manual rule dataset support uses synthetic field samples and records `is_production_evaluation=false`. The successful manual rule run used two synthetic `PROC_AMOUNT_001` samples with no failed cases; `explainability_rate=0.5` is expected because only the fail sample requires evidence. It validates amount consistency without DB evidence IDs, is not a production-scale rule evaluation, and does not exercise the full DB task/document/field Rule Engine workflow.
Manual RAG dataset support uses synthetic inline documents and records `is_production_evaluation=false`. The successful manual RAG run used two synthetic samples with no failed cases and `1.0` answer/citation/no-answer metrics. Citations are generated only from sample `input.documents` document IDs, and the runner does not call real embedding, rerank, or answer Providers. It is not production-scale RAG evaluation and does not exercise persistent vector-store, four-library isolation, or workpaper-scope retrieval.
Manual Agent dataset support uses synthetic workflow contract samples and records `is_production_evaluation=false`. The successful manual Agent run used two synthetic samples with no failed cases and passed workflow, required-tool, forbidden-tool, review-routing, conclusion-guardrail, and final-status metrics. The runner simulates tool constraints, review routing, conclusion guardrails, and final status without creating `agent_runs` / `agent_steps` or calling real AgentService workflow. It is not production-scale Agent evaluation and does not exercise retry, Review Center, Report Center, or DB workflow integration.
Manual E2E dataset support uses synthetic workflow contract samples and records `is_production_evaluation=false`. The successful manual E2E run used one synthetic procurement walkthrough sample with no failed cases and `1.0` step, document, business-key, rule, report, evidence, and guardrail metrics. The runner simulates procurement walkthrough steps, document classification, key extraction, `PROC_AMOUNT_001`, report-generation flag, evidence-index flag, and high-risk guardrail without creating real tasks, documents, reports, DB IDs, uploaded files, or local report files. It is not production-scale E2E evaluation and does not exercise the full DB/API workflow.
Manual regression dataset support records `is_production_evaluation=false` and aggregates ten manual dataset runners: OCR, classification, extraction, rule, RAG, Agent, persistent RAG workflow, Agent DB workflow, E2E, and full DB workflow. It blocks recursive `regression` execution and reports per-eval sample counts, failed cases, pass rates, dataset-driven flags, and production flags. It is not production-scale regression evaluation and does not replace real/desensitized large regression datasets or real Provider quality evidence.
